ajax
异步刷新
1、由前端发送请求给后台，然后前端页面保留（页面无跳转，同步刷新页面是有跳转）
2、后台接收数据处理，并把结果返回给前端
3、前端接收后台数据后，根据结果处理

利用jquery
1、由前端发送请求给后台，然后前端页面保留（页面无跳转，同步刷新页面是有跳转）
1.1、找到事件的促发点
点击页码时->为页码的组件绑定一个点击事件
$("#pageDiv a").click(function(){
	
});
可以在function中写入一段代码测试一下
1.2、阻止页码跳转
在function最后加入一句return false;在任何组件绑定的时间中，如果出现return false就表示返回的结果为假，中断组件后续的所有操作
1.3、写入ajax的发送配置->发送什么给后台处理->借组jq的ajax方法（从手册上面找例子）
$("#pageDiv a").click(function(){
	//this：指向当前对象->谁点击谁促发谁是当前对象
	//attr：获取对象标签任何的属性
	var URL = $(this).attr('href');
	//alert(URL);
	//return false;
	$.ajax({
		type: "GET",//参数发送方式->根据原本的提交时get还是post
		url: URL,//后台地址，通常后台是不用写死的，因为本身的应用就已经会包含原本的地址，基本上有方法可以提取
		//data: "p=3&m=news&c=index&admin=1",//传输的数据，get方式传参是可以省略，因为参数都在地址那里
		success: function(msg){
			alert( "Data Saved: " + msg );
		}
	});
	return false;//阻止页码跳转
});
ajax的第一过程已经完成
2、后台接收数据处理，并把结果返回给前端
在一般ajax开发过程中，后台的代码基本可以不变。因为在完成ajax之前，你应该先完成一个后台刷新的版本（易维护，易开发，易排错）

3、前端接收后台数据后，根据结果处理
3.1、前端如何接受数据
success: function(msg){
	alert(msg);
}//当ajax成功调用（1,2正确），通过此函数获取后台的参数（msg：后台的输出），并且在此函数中处理结果
理论此时ajax已经成功的，返回的数据，由于没有经过任何处理，因此后台返回本身是视图的html，接收到也是完整的html，但这个数据是完全不符要求->我们要的是改变的部分
3.2、需要处理后台的数据（改变后台输出的部分）
3.2.1、如何得到需要改变的部分->列表数组->变为json
3.2.2、面临2个问题：
1、json能直接用吗？不能，因为在php输出的json，js仍当作是一个普通的字符串。所以必须把“字符串”变成js的对象
dataType:"json",//规定后台返回的数据格式必须是json，否则出错，而接收到的数据，直接就是可用的json。该属性一定要放到ajax确认接收参数无误后才加上
2、直接访问网页怎么办？如何区分现在是ajax访问(直接访问与ajax访问)
!empty($_SERVER['HTTP_X_REQUESTED_WITH'])
[HTTP_X_REQUESTED_WITH] => XMLHttpRequest ->现在通过ajax访问

3.3、根据结果处理
通过json，组装出新的列表，并且替换原本的html
